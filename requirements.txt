# requirements.txt kamu yang lama cuma nge-pin package yang kamu
# import LANGSUNG (streamlit, pandas, numpy, dst). Semua dependency
# TURUNAN-nya (cryptography, cffi, pyarrow, curl_cffi, pydantic-core,
# protobuf, rpds-py, dan puluhan lainnya) DIBIARKAN BEBAS -- artinya
# tiap kali Streamlit Cloud "wake up" app kamu, dia install ulang
# dari nol (lihat log kamu: "Processing dependencies..." muncul
# LAGI tiap boot), dan pip/uv boleh narik versi TERBARU yang ada
# hari itu untuk semua package turunan itu.
#
# Kalau salah satu package native (C extension) itu baru rilis
# versi yang ada bug/incompatible sama versi lain (ABI mismatch),
# proses Python bisa langsung mati "Segmentation fault" -- PAS di
# awal boot, sebelum sempat render apa-apa. Ini yang paling masuk
# akal menjelaskan kenapa crash-nya "abis reboot" dan KADANG jalan
# KADANG enggak: kombinasi versi yang ke-install emang beda-beda
# tiap kali di-install ulang.
#
# File ini MENGUNCI SEMUA package (termasuk turunan) ke versi
# persis yang berhasil TER-INSTALL di log terakhirmu (proses
# install-nya sendiri sukses, baru crash belakangan pas script
# jalan) -- supaya tiap reboot/wake-up dapat environment yang
# 100% IDENTIK, bukan tebak-tebakan versi baru tiap hari.
#
# CATATAN: kalau setelah pakai file ini crash MASIH terjadi,
# artinya kombinasi versi ini sendiri yang bermasalah (bukan
# soal "versi baru tiba-tiba muncul") -- lihat catatan SAFE_MODE
# di ai_idx_trading_terminal.py untuk uji coba lanjutan (menonaktifkan
# EncryptedCookieManager sementara, karena itu satu-satunya kode
# native/crypto yang jalan di SETIAP render halaman).
